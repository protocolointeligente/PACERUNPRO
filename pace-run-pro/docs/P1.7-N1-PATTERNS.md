# P1.7 — N+1 Query Patterns: Detecção e Correção

## Implementação Completa ✅

### O que é N+1?
**N+1 problem**: 1 query para buscar lista + N queries (uma para cada item) = N+1 queries ao invés de 1.

**Exemplo:**
```typescript
// ❌ N+1: 1 + 5 = 6 queries
const athletes = await prisma.athlete.findMany();
const result = [];
for (const athlete of athletes) {
  const workouts = await prisma.workout.findMany({ where: { athleteId: athlete.id } });
  result.push({ ...athlete, workouts });
}

// ✅ Otimizado: 1 query
const athletesWithWorkouts = await prisma.athlete.findMany({
  include: {
    trainingPlans: {
      include: { weeks: { include: { workouts: true } } }
    }
  }
});
```

## Ferramentas Implementadas

### 1. **N1Detector** (`src/lib/n1-detector.ts`)
Detecta padrões N+1 em métricas de query capturadas pelo P1.6 QueryAnalyzer.

**Características:**
- Agrupa queries por modelo + ação
- Calcula severidade (critical → high → medium → low)
- Fornece recomendações específicas
- Estima ganho de performance (até 80% redução)

**Uso:**
```typescript
import { N1Detector } from '@/lib/n1-detector';
import { getQueryAnalyzer } from '@/lib/query-analyzer';

const analyzer = getQueryAnalyzer();
const metrics = analyzer.getMetrics();
const detection = N1Detector.detect(metrics);

console.log(N1Detector.formatReport(detection));
// 🟠 Athlete.findMany (high)
//    Execuções: 24
//    Duração total: 2400ms
//    Recomendação: Use 'include' ou 'select' para carregar relacionamentos
```

### 2. **Coach Queries Optimized** (`src/lib/coach-queries-optimized.ts`)
Funções refatoradas que eliminam N+1 em operações comuns.

**Funções Disponíveis:**
- `getCoachAthletesOptimized()` - 1 query ao invés de N
- `getCoachPlansOptimized()` - Planos com select eficiente
- `getCoachWeeklyDataOptimized()` - Dados de semana com todas as relações
- `getCoachActionCenterOptimized()` - Estatísticas paralelas (Promise.all)
- `getCoachGroupsOptimized()` - Grupos com members + athletes
- `getCoachLeadsOptimized()` - Leads com filtros

### 3. **Padrões N+1** (`src/lib/n1-patterns.ts`)
Documentação de padrões comuns e suas correções.

## Checklist: Como Usar P1.7

### Para Desenvolvedores
- ✅ Use `include`/`select` para carregar relacionamentos
- ✅ Batch múltiplas queries independentes com `Promise.all`
- ✅ Use `{ in: [...] }` em `findMany` ao invés de loop `findUnique`
- ✅ Selecione apenas campos necessários
- ✅ Revise code novo com QueryAnalyzer + N1Detector
