### 1. N1Detector Utility (`src/lib/n1-detector.ts`)
**Arquivo criado:** 250+ linhas de código production-ready
**Funcionalidades:**
- `N1Detector.detect()` - Detecta padrões N+1 em métricas de query
- `calculateSeverity()` - Classifica severidade (critical → low)
- `getRecommendation()` - Fornece dicas específicas de refactoring
- `estimateImprovement()` - Estima ganho de performance
- `formatReport()` - Formata relatório legível com ícones de severidade

**Exemplo de Uso:**
```typescript
import { N1Detector } from '@/lib/n1-detector';
import { getQueryAnalyzer } from '@/lib/query-analyzer';

const analyzer = getQueryAnalyzer();
const detection = N1Detector.detect(analyzer.getMetrics());
console.log(N1Detector.formatReport(detection));
```

## 🔄 Integração com P1.6

P1.7 é construído sobre P1.6:
- **P1.6** coleta métricas de todas as queries (queryPerformanceMiddleware)
- **P1.7** analisa essas métricas para detectar padrões N+1
- **Flow:** Query executada → P1.6 registra → P1.7 detecta padrão

## 🚀 Como Usar em Produção

### Opção 1: Usar Funções Prontas
```typescript
import { getCoachAthletesOptimized } from '@/lib/coach-queries-optimized';

const athletes = await getCoachAthletesOptimized(coachUserId);
```

### Opção 2: Refatorar Manualmente
Seguir padrões em `src/lib/n1-patterns.ts`:
1. Adicionar `include` para relações 1:many
2. Usar `select` para limitar campos
3. Agrupar queries paralelas com `Promise.all()`
4. Usar `findMany({ where: { id: { in: [...] } } })` para múltiplos IDs

### Opção 3: Detectar Automaticamente
```typescript
const detector = new N1Detector();
const patterns = detector.detect(metrics, 100); // 100ms window
console.log(detector.formatReport(patterns));
```
