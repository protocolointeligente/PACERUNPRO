Roteiro por Fases

0
Semanas 1–2
Ligar os Motores
Configurações técnicas — faça uma vez, não precisa mais mexer
  • 1
    Criar conta no Neon (banco de dados gratuito) ⏱ 10 min
    • Acesse neon.tech → Create account (pode usar Google)
    • Clique "New Project" → nome: pacerunpro → região: AWS us-east-1
    • Copie a connection string (começa com postgresql://...)
    • Guarde essa string em local seguro — você vai usar no próximo passo
  • 2
    Configurar o banco no Vercel ⏱ 15 min
    • Acesse vercel.com → seu projeto → Settings → Environment Variables
    • Adicione: DATABASE_URL = [a connection string do Neon]
    • Adicione: NEXTAUTH_SECRET = [qualquer texto longo, ex: uma frase com espaços]
    • Adicione: NEXTAUTH_URL = https://www.pacerunpro.com.br
    • Clique Save e depois Redeploy para aplicar
  • 3
    Criar as tabelas no banco (migration) ⏱ 5 min (com suporte técnico)
    • Abra o terminal no Claude Code (ou peça ao desenvolvedor)
    • Execute: npx prisma migrate deploy
    • Depois: npx prisma db seed
    • Isso cria o usuário administrador inicial automaticamente
  • 4
    Configurar login com Google ⏱ 30 min
    • Acesse console.cloud.google.com
    • Crie um projeto → APIs & Services → Credentials → OAuth 2.0 → Web application
    • Em "Authorized redirect URIs" adicione: https://www.pacerunpro.com.br/api/auth/callback/google
    • Copie o Client ID e Client Secret para as env vars do Vercel
    • Dica: siga um tutorial em vídeo no YouTube pesquisando "NextAuth Google OAuth"
  • 5
    Apontar o domínio para o Vercel ⏱ 10 min + até 24h propagação
    • No registrador do domínio (onde registrou pacerunpro.com.br): adicione registro CNAME apontando para cname.vercel-dns.com
    • No Vercel: Settings → Domains → adicione www.pacerunpro.com.br
    • Aguarde a propagação: pode levar até 24h, normalmente bem menos
    • Valide em: dnschecker.org
✅
Resultado desta fase

Sistema no ar com login funcionando, dados salvos de verdade no banco. Você tem uma URL real que qualquer pessoa pode acessar.

1
Semanas 2–3
Começar a Cobrar
Integração de pagamentos — Pix, boleto e cartão
  • 1
    Criar conta no Asaas (plataforma de pagamentos BR) ⏱ 24–48h aprovação
    • Acesse asaas.com → Abrir conta grátis
    • Envie os documentos (CPF ou CNPJ) — aprovação em 24–48 horas
    • Crie os planos de assinatura: Starter R$149 / Growth R$349 / Business R$699
    • Navegue até Configurações → Integração e copie sua API Key
  • 2
    Configurar os planos no sistema ⏱ 4–8h com desenvolvedor
    • Adicione ASAAS_API_KEY nas environment variables do Vercel
    • A página /checkout já existe no sistema — precisa ser conectada à API do Asaas
    • Instrução ao desenvolvedor: "Conectar o checkout com Asaas e liberar acesso automaticamente após pagamento confirmado"
    • Configure também o webhook do Asaas para notificar pagamentos
  • 3
    Testar o fluxo completo ⏱ 30 min
    • Crie uma conta de teste com e-mail diferente do seu
    • Faça um pagamento real de R$1,00 (para validar todo o fluxo)
    • Confirme que o acesso foi liberado automaticamente ao plano
    • Confirme que chegou e-mail de confirmação de pagamento
    • Teste também com Pix — é o método mais usado no Brasil
✅
Resultado desta fase

Primeiros pagamentos reais sendo processados automaticamente. Você recebe o dinheiro, o cliente acessa a plataforma — sem intervenção manual.

2
Semana 3
Comunicação Automática
E-mails que trabalham por você enquanto você dorme
  • 1
    Criar conta no Resend (e-mail transacional) ⏱ 20 min
    • Acesse resend.com → criar conta grátis (até 3.000 e-mails/mês sem custo)
    • Verificar o domínio pacerunpro.com.br: adicionar registro DNS TXT fornecido pelo Resend
    • Aguarde a verificação (geralmente menos de 1 hora)
    • Copie a API Key para usar no próximo passo
  • 2
    Configurar os e-mails automáticos ⏱ 4–6h com desenvolvedor
    E-mails essenciais a implementar:
    • Boas-vindas — disparado automaticamente após o cadastro
    • Confirmação de pagamento — após cada pagamento aprovado
    • Lembrete de renovação — 3 dias antes do vencimento
    • Senha esquecida — link de redefinição seguro

    Instrução ao desenvolvedor: "Integrar Resend nos eventos de cadastro e pagamento usando os templates aprovados"
  • 3
    Testar cada e-mail ⏱ 15 min
    • Faça um cadastro de teste e confirme que chegou o e-mail de boas-vindas
    • Simule um pagamento e confirme que chegou a confirmação
    • Verifique se os e-mails não estão caindo na pasta de spam
    • Teste em Gmail, Outlook e e-mail corporativo se possível
✅
Resultado desta fase

Toda comunicação com clientes é automática, profissional e com a identidade visual do PACE RUN PRO. Você foca em vender, o sistema cuida do relacionamento.

3
Mês 1
Validação com Beta-testers
5 treinadores de confiança que vão te dar feedback real
  • 1
    Selecionar 5 treinadores parceiros
    • Critérios: já te conhecem, têm pelo menos 10 atletas ativos, são organizados
    • Oferta: 3 meses gratuitos em troca de feedback honesto e depoimento
    • Crie um grupo de WhatsApp exclusivo chamado "PACE PRO Beta 🚀"
    • Documente quem são, quantos atletas têm e qual o maior problema deles hoje
  • 2
    Onboarding personalizado
    • Marque uma videochamada de 30 minutos com cada treinador individualmente
    • Mostre as principais funcionalidades (use este manual como guia)
    • Cadastre manualmente os primeiros 5–10 atletas de cada um para facilitar o início
    • Deixe claro o que você espera deles em troca (feedback + depoimento)
  • 3
    Coletar feedback estruturado
    Faça check-ins regulares com perguntas específicas:
    • Semana 2: "O que está faltando para você usar todos os dias?"
    • Semana 4: "O que você mais usa? O que nunca usou?"
    • Semana 6: "Você pagaria por isso? Quanto acha justo?"
    • Priorize melhorias que apareçam na fala de 3 ou mais pessoas
  • 4
    Gerar prova social
    • Peça um depoimento em vídeo de 1 minuto (pelo WhatsApp mesmo, qualidade não importa)
    • Peça também uma avaliação escrita de 2–3 linhas para publicar no site
    • Documente resultados concretos: "X atletas gerenciados", "Y% de aderência melhorada"
    • Guarde esses materiais — serão a base de toda sua comunicação de vendas
✅
Resultado desta fase

Validação real do produto com usuários reais, primeiros casos de sucesso documentados e prova social concreta para usar nas vendas.

Mês 2
Lançamento Comercial
Hora de vender de verdade
  • 1
    Definir e publicar os preços
    • Atualize a landing page com os preços definitivos (validados com os beta-testers)
    • Crie a página /precos com comparativo visual entre os planos
    • Adicione botão de trial de 14 dias — converte significativamente melhor que venda direta
    • Inclua depoimentos dos beta-testers logo abaixo da tabela de preços
  • 2
    Criar presença digital
    • Instagram @pacerunpro: configurar perfil profissional com foto, bio e link
    • Prepare 9 posts antes de lançar (feed visualmente coeso desde o primeiro dia)
    • Mix de conteúdo: bastidores do sistema, funcionalidade em vídeo (Reels), dica de corrida
    • Bio sugerida: "A plataforma #1 para assessorias de corrida 🚀 | Gerencie atletas, prescreva treinos, cresça 👇"
  • 3
    Campanha de lançamento
    • Publique Stories com countdown nos 7 dias antes do lançamento
    • Faça uma Live no Instagram apresentando o sistema ao vivo
    • Oferta de lançamento: 50% off no primeiro mês para as 20 primeiras assessorias
    • Crie urgência real: quando os 20 slots acabarem, a oferta desaparece
  • 4
    Abordagem direta — funciona melhor que anúncios no início Meta: 10 demos → 3–5 vendas
    • Liste 50 assessorias do seu estado no Instagram (busque por hashtags de corrida)
    • Envie DM personalizada: "Oi [nome], sou o Ricardo. Criei uma plataforma específica para assessorias de corrida. Posso te mostrar em 15 minutos esta semana?"
    • Não faça copy-paste genérico — mencione algo do perfil deles para personalizar
    • Acompanhe respostas numa planilha simples (nome, status, data de contato)
✅
Resultado desta fase

Primeiros 5–10 clientes pagantes. Você tem receita real, casos de sucesso documentados e um processo de venda que funciona.

5
Meses 3–6
Crescimento
Escalar o que está funcionando
  • 1
    Tráfego pago no Meta Ads R$ 500–1.000/mês
    • Público: interesse em "Corrida de rua" + "Personal trainer" + Brasil
    • Criativo: vídeo de tela do sistema com narração de 30 segundos
    • Destino: página de cadastro com trial grátis de 14 dias
    • Só comece ads quando tiver pelo menos 5 depoimentos — eles aumentam muito a conversão
  • 2
    Programa de indicação
    • Mecanismo simples: "Indique um treinador e ganhe 1 mês grátis"
    • Cada cliente satisfeito tem potencial de trazer 2–3 novos clientes da rede dele
    • Solicite ao desenvolvedor: uma página simples de indicação com link único por cliente
    • Comunique o programa por e-mail para toda a base de clientes ativos
  • 3
    Parcerias estratégicas
    • Fabricantes de tênis (New Balance, Asics, Brooks) — patrocínio de funcionalidade ou co-marketing
    • Nutricionistas esportivos — indicação mútua (você indica nutricionistas, eles indicam o sistema)
    • Grupos e clubes de corrida — desconto corporativo para assessorias que atendem o grupo
    • Para cada parceria: formalize por e-mail, com regras claras e benefício mútuo
  • 4
    Marketing de conteúdo (retenção + aquisição)
    • Instagram: 1 post/dia (mistura de sistema + dicas de corrida)
    • Reels: 1 por semana mostrando uma funcionalidade em 30–60 segundos
    • Newsletter mensal: para clientes ativos — novidades, dicas, cases de sucesso
    • Objetivo duplo: atrair novos clientes E manter os atuais engajados (reduz churn)
  • 5
    Expansão de funcionalidades
    Prioridades baseadas em demanda dos beta-testers:
    • Sync com Garmin/Strava (integração com relógios — muito pedida)
    • App nativo iOS/Android (para atletas acompanharem treinos no celular)
    • Relatório automático por e-mail semanal para atletas
    • Construa em ordem de demanda — só desenvolva o que clientes pedem repetidamente
✅
Resultado desta fase

Base crescendo de forma sustentável, múltiplos canais de aquisição ativos e produto evoluindo com base em feedback real do mercado.
