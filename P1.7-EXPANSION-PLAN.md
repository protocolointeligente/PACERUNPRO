## 3. Padrões Reutilizáveis

### Padrão A: Nested Relationship Flattening

**Problema:**
```typescript
where: {
  athlete: {
    coach: {
      userId: session.user.id
    }
  }
}
```

**Solução:**
```typescript
// 1. Fetch coach ID
const coach = await prisma.coach.findUnique({
  where: { userId: session.user.id },
  select: { id: true }
});

// 2. Use coach.id in WHERE
where: {
  athlete: {
    coachId: coach.id
  }
}
```

---

### Padrão B: Loop-Based N+1 Prevention

**Problema:**
```typescript
for (const workout of workouts) {
  const exercises = await prisma.exercise.findMany({
    where: { workoutId: workout.id }
  });  // ← N queries!
}
```

**Solução:**
```typescript
// Batch fetch all at once
const workoutIds = workouts.map(w => w.id);
const allExercises = await prisma.exercise.findMany({
  where: { workoutId: { in: workoutIds } }
});

// Map results
const exercisesByWorkout = new Map();
allExercises.forEach(ex => {
  if (!exercisesByWorkout.has(ex.workoutId)) {
    exercisesByWorkout.set(ex.workoutId, []);
  }
  exercisesByWorkout.get(ex.workoutId).push(ex);
});

// Use mapped data
const result = workouts.map(w => ({
  ...w,
  exercises: exercisesByWorkout.get(w.id) ?? []
}));
```
